### Runs API Hooks (`src/api/runs.ts`)


## Backend API Surface (runs)

### GET /api/runs

- Params: `pipeline_name?`, `status?`, `started_after?`, `started_before?`, `offset?` (default 0), `limit?` (default 50, max 200)
- Returns: `{ items: RunListItem[], total: number, offset: number, limit: number }`
- Ordered by `started_at` DESC

### GET /api/runs/{run_id}

- Returns: `RunDetail` with embedded `steps: StepSummary[]`

### POST /api/runs

- Body: `{ pipeline_name: string }`
- Returns: `{ run_id: string, status: "accepted" }` (202)

## Key Implementation Decisions

### 1. Status Filter Bridging

- `status` is in URL search params (already defined in index route schema)
- `pipelineName`, `startedAfter`, `startedBefore` are in Zustand `useFiltersStore`
- RunList component must merge both sources into `RunListParams` for `useRuns()`

### 2. Pagination Model

- URL has `page` (1-indexed), API uses `offset` (0-indexed)
- Conversion: `offset = (page - 1) * PAGE_SIZE`
- Default page size: recommend 25 (UI-friendly) or 50 (backend default)
- Total pages: `Math.ceil(data.total / PAGE_SIZE)`
- Navigate pages by updating URL search param: `navigate({ search: { page: newPage } })`

### 3. shadcn Components Needed

- `Table` (table, thead, tbody, tr, th, td primitives)
- `Badge` for status pills
- `Button` for pagination controls
- `Select` for filter dropdowns
- `Tooltip` for full timestamps and full run IDs
- Install: `npx shadcn@latest add table badge button select tooltip`

### 4. Component File Structure

```
src/
  components/
    ui/          # shadcn primitives (auto-placed by shadcn CLI)
    runs/
      RunTable.tsx       # table body with rows
      RunRow.tsx         # single row with status badge, relative time
      FilterBar.tsx      # pipeline + status + date filters
      Pagination.tsx     # page navigation controls
      StatusBadge.tsx    # color-coded status indicator
```

### 5. Status Color Mapping

| Status | Color (dark theme) | Tailwind class |
|---|---|---|
| running | blue | `text-blue-400 bg-blue-400/10` |
| completed | green | `text-green-400 bg-green-400/10` |
| failed | red/destructive | `text-destructive` |
| pending | yellow/amber | `text-amber-400 bg-amber-400/10` |
| (unknown) | gray/muted | `text-muted-foreground` |

### 6. Relative Timestamps

- Use built-in `Intl.RelativeTimeFormat` or a lightweight helper (no dayjs/date-fns dep needed)
- Show: "2m ago", "1h ago", "3d ago"
- Tooltip: full ISO datetime string

### 7. Run ID Display

- Truncate to 8 chars: `run.run_id.slice(0, 8)`
- Full ID visible on hover via Tooltip
- Monospace font for readability
