## 3. Routing Setup

- **TanStack Router v1.161.3** with file-based routing via `@tanstack/router-plugin/vite`
- Auto-generates `routeTree.gen.ts` from `src/routes/` directory structure
- `autoCodeSplitting: true` enabled -- route components are automatically lazy-loaded
- Routes register via `createFileRoute('/path')({ component, validateSearch? })`
- Search params validated with `@tanstack/zod-adapter` + Zod schemas
- The `/live` route already exists as a placeholder -- will be replaced in-place

## 4. State Management

### Zustand Stores
- **ui.ts**: `sidebarCollapsed`, `theme`, `selectedStepId`, `stepDetailOpen` -- persisted to localStorage (sidebar + theme only)
- **filters.ts**: `pipelineName`, `startedAfter`, `startedBefore` -- ephemeral, resets on reload
- **websocket.ts**: `WsConnectionStatus` union (`idle|connecting|connected|replaying|closed|error`), `error`, `reconnectCount` -- ephemeral

### TanStack Query
- `queryClient` at `src/queryClient.ts` with 30s default staleTime, retry: 2
- All hooks use hierarchical `queryKeys` factory for targeted invalidation
- Dynamic `staleTime: Infinity` for terminal runs (immutable data), shorter staleTime + polling for active runs

## 5. WebSocket Infrastructure (from Task 31)

### Backend
- Endpoint: `ws://host/ws/runs/{run_id}` (no /api prefix)
- Vite proxy: `/ws` -> `ws://localhost:8642`
- Three server behaviors: (1) not found (close 4004), (2) replay persisted events + replay_complete, (3) live stream + stream_complete

### Frontend Hook: `useWebSocket(runId: string | null)`
- Location: `src/api/websocket.ts`
- Connects to `/ws/runs/{runId}`, handles all 3 server behaviors
- Pipeline events appended to TanStack Query event cache via `queryClient.setQueryData`
- Step-scoped events trigger `queryClient.invalidateQueries` on steps list
- `stream_complete` invalidates run detail query
- Exponential backoff reconnect (1s base, 30s max, capped)
- React 19 Strict Mode guard (skips duplicate connections)
- Returns `{ status, error }` from Zustand websocket store

### WsMessage Types (discriminated union)
- `WsHeartbeat`, `WsStreamComplete`, `WsReplayComplete`, `WsError`, `WsPipelineEvent`
- `parseWsMessage()` tags raw pipeline events with `type: 'pipeline_event'`

## 7. Existing API Hooks for Live View

### Pipeline Selection
- `usePipelines()` -> `{ pipelines: PipelineListItem[] }` -- list all registered pipelines
- `usePipeline(name)` -> `PipelineMetadata` with strategies, steps, execution_order

### Run Triggering
- `useCreateRun()` -> mutation, sends `TriggerRunRequest { pipeline_name }`, returns `{ run_id, status }`
- **Note**: `TriggerRunRequest` only has `pipeline_name` field currently. Task 31 summary noted `input_data` extension needed for task 37.

### Live Run Data
- `useRun(runId)` -> run detail with polling for active runs
- `useSteps(runId, runStatus?)` -> step list with polling for active runs
- `useEvents(runId, filters?, runStatus?)` -> event list with polling for active runs
- `useWebSocket(runId)` -> WS connection with cache integration

## 11. Existing Patterns to Follow

### Route File Pattern
```typescript
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/live')({
  component: LivePage,
  // validateSearch: zodValidator(schema),  -- if search params needed
})

function LivePage() { ... }
```

### Component Pattern
- Named function exports (not arrow/default)
- Props interfaces defined above component
- Loading skeletons as separate private functions
- Error/empty states as separate private functions

### Hook Consumption Pattern (from $runId.tsx)
- Compose multiple hooks at top of component
- Use `useMemo` for derived data (e.g., `deriveStepStatus`)
- Pass `run?.status` to downstream hooks for dynamic staleTime
- UI state from Zustand (selectStep, closeStepDetail)

### Styling Pattern
- Tailwind utility classes only (no CSS modules, no CSS-in-JS)
- `cn()` for conditional class merging
- shadcn/ui components for common UI elements
- Dark theme as default (`document.documentElement.classList.add('dark')`)
- CSS variables for theme colors (oklch color space)

## 14. Risk Assessment

| Risk | Mitigation |
|------|-----------|
| EventStream performance with high event volume | Virtualized list (react-window) if > 1000 events, else simple div |
| StepDetailPanel in live context (step data incomplete during execution) | StepDetailPanel already handles loading/error states; useStep with non-terminal staleTime polls |
| Column layout responsiveness | Tailwind responsive breakpoints; collapse to single column on mobile |
| Backend TriggerRunRequest lacks input_data | OK for task 37; task 38 extends both backend and frontend together |
