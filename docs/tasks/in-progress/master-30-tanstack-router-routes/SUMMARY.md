## Issues Encountered

### Review Fix 1: Inconsistent search param schema pattern in index.tsx
`index.tsx` initially used `fallback().optional()` while `$runId.tsx` used `fallback().default()`. Context7 docs and the review identified `.default()` as the correct pattern, giving deterministic values instead of `undefined` when params are absent.
**Resolution:** Changed `index.tsx` schema to `fallback(z.number().int().min(1), 1).default(1)` and `fallback(z.string(), '').default('')`.

### Review Fix 1: min-h-screen ineffective inside root layout
`index.tsx` used `min-h-screen` (100vh) inside `__root.tsx`'s `<main className="flex-1 overflow-auto">` constrained by `h-screen`. The class created a 100vh scrollable region inside the main area rather than filling it.
**Resolution:** Changed to `min-h-full`.

### Review Fix 2: Unused `search` variable broke type-check
The fix commit for the schema issue introduced `const search = Route.useSearch()` but `search` was never read. `tsconfig.app.json` has `noUnusedLocals: true`, causing `tsc -b --noEmit` to fail with TS6133.
**Resolution:** Removed the unused variable entirely; task 31 will introduce it when wiring up API hooks.

### Accepted: Zod 4 peer dependency mismatch
`@tanstack/zod-adapter@1.161.3` declares `peerDependencies: {"zod": "^3.23.8"}` but `zod@4.3.6` is installed. Runtime testing confirmed all used APIs (`z.object`, `z.string`, `z.number`, `fallback`) work correctly. Zod 4 ships a `zod/v3` compat layer.
**Resolution:** Accepted as-is. Monitor when upgrading `@tanstack/zod-adapter`; it will likely update its peer dep range to include zod 4.
