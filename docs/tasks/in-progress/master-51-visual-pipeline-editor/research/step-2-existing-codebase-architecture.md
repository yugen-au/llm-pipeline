## 2. Routing System

- **Library**: TanStack Router v1.161.3
- **Mode**: File-based routing via `@tanstack/router-plugin/vite` with `autoCodeSplitting: true`
- **Route definition pattern**:
  ```tsx
  export const Route = createFileRoute('/path')({
    validateSearch: zodValidator(schema),  // optional
    component: PageComponent,
  })
  ```
- **Search params**: Zod schemas + `zodValidator()` from `@tanstack/zod-adapter` + `fallback()` for defaults
- **Auto-gen**: `routeTree.gen.ts` regenerated on file changes
- **Type registration**: `router.ts` declares module augmentation for `Register.router`

### Current Routes

| File | Path | Layout Pattern |
|------|------|---------------|
| `__root.tsx` | - | Sidebar + main area |
| `index.tsx` | `/` | Single scrollable column |
| `live.tsx` | `/live` | 3-column desktop / tabs mobile |
| `pipelines.tsx` | `/pipelines` | 2-panel left-right with `?pipeline=` search param |
| `prompts.tsx` | `/prompts` | 2-panel left-right with `?key=` search param |
| `creator.tsx` | `/creator` | 3-column desktop / tabs mobile |
| `runs/$runId.tsx` | `/runs/:runId` | Detail view |

### Adding a New Route

To add `/editor`:
1. Create `src/routes/editor.tsx` with `createFileRoute('/editor')`
2. `routeTree.gen.ts` auto-regenerates
3. Add nav entry to `Sidebar.tsx` navItems array

## 4. Component Patterns & Conventions

### Function Style

- Named function declarations (NOT arrow const)
- Props interface immediately above component:
  ```tsx
  interface ComponentNameProps {
    prop1: string
    prop2: () => void
  }

  export function ComponentName({ prop1, prop2 }: ComponentNameProps) {
    ...
  }
  ```

### Import Conventions

- Path alias: `@/` maps to `src/`
- UI primitives: `@/components/ui/...`
- Domain components: `@/components/{domain}/...`
- Shared components: `@/components/shared` (barrel export)
- API hooks: `@/api/{domain}`
- Types: `@/api/types`
- Utilities: `@/lib/utils`
- Stores: `@/stores/{domain}`

### State Patterns

- **Loading**: Skeleton divs with `animate-pulse bg-muted` + fixed width
- **Error**: Centered `text-sm text-destructive` message
- **Empty**: Centered `text-muted-foreground` message (or `<EmptyState message="..." />`)
- **Data guard**: `if (isLoading) return <Skeleton />`; `if (error || !data) return <Error />`

### Component Composition

- Route files contain the page component + route definition
- Complex pages extract column content into domain components
- Components are domain-grouped: `components/creator/`, `components/pipelines/`, etc.
- Shared reusable patterns go to `components/shared/`
- Each component file has colocated `.test.tsx`

## 5. API & Data Fetching Layer

### API Client

```tsx
// api/client.ts
export async function apiClient<T>(path: string, options?: RequestInit): Promise<T>
```
- Prepends `/api` to path
- Throws typed `ApiError(status, detail)` on non-OK
- Vite dev server proxies `/api` -> `http://localhost:8642`

### TanStack Query

- **Version**: 5.90.21
- **QueryClient**: staleTime 30s, retry 2, refetchOnWindowFocus false
- **Query keys**: Hierarchical factory in `query-keys.ts`
- **Pattern**: Custom hooks wrap `useQuery()` / `useMutation()` with typed keys + fns
- **Conditional fetch**: `enabled: Boolean(value)` for optional params
- **Cache invalidation**: `queryClient.invalidateQueries({ queryKey: queryKeys.domain.all })`

### Query Key Factory

```tsx
export const queryKeys = {
  runs: { all, list(filters), detail(id), context(id), steps(id), step(id,n), events(id,filters) },
  prompts: { all, list(filters), detail(key) },
  pipelines: { all, detail(name), stepPrompts(name, stepName) },
  creator: { all, drafts(), draft(id) },
}
```

For the visual editor, a new `editor` section will be needed:
```tsx
editor: {
  all: ['editor'] as const,
  draftPipelines: () => ['editor', 'draft-pipelines'] as const,
  draftPipeline: (id: number) => ['editor', 'draft-pipelines', id] as const,
}
```

## 6. State Management

### Zustand Stores

| Store | File | Persisted | State |
|-------|------|-----------|-------|
| `useFiltersStore` | `stores/filters.ts` | No | Run list filters (pipelineName, dateRange) |
| `useUIStore` | `stores/ui.ts` | Partial | sidebarCollapsed, theme (persisted); selectedStepId, stepDetailOpen (ephemeral) |
| `useWsStore` | `stores/websocket.ts` | No | WS status, error, reconnectCount, latestRun |

### Zustand Pattern

```tsx
export const useMyStore = create<MyState>()(
  devtools(
    persist?( // optional
      (set) => ({
        field: initialValue,
        action: () => set({ field: newValue }),
      }),
      {
        name: 'storage-key',
        partialize: (state) => ({ /* only persisted fields */ }),
      },
    ),
    { name: 'store-name', enabled: import.meta.env.DEV },
  ),
)
```

### State Location Decisions

- **URL search params**: Shareable state (selected pipeline, selected prompt key)
- **Local React state**: Form state, workflow state machines
- **Zustand store**: Cross-component ephemeral state, persisted preferences
- **TanStack Query cache**: Server state, API responses

For the visual editor, likely state locations:
- **URL search params**: `?draft=<draftPipelineId>` for selected draft pipeline
- **Local state**: Step order, drag state, form values for step properties
- **Zustand store**: Only if cross-component state needed (unlikely for initial impl)
- **TanStack Query**: Draft pipeline CRUD queries/mutations
